# [작업 지시서] 공유창고 무인 통합 관리 시스템 자체 구축

> 외주 개발사나 개발자에게 바로 전달하여 계약 및 개발을 진행할 수 있도록 작성된 **[공유창고 무인 통합 관리 시스템 개발 작업 지시서(RFP)]**입니다.
> 자체 프로그램 개발 및 네이버 예약 우회 연동(API 미사용), 윈도우 미니 PC 환경의 특성을 모두 반영하여 명확하게 작성했습니다.

---

## 1. 프로젝트 개요

- **프로젝트명:** 윈도우 미니 PC 기반 공유창고 무인 키오스크 및 하드웨어 제어 시스템 구축
- **개발 목적:** 상주 인력 없이 24시간 운영 가능한 공유창고 관리 프로그램을 자체 개발하며, 현장 키오스크 결제, 하드웨어(출입문/캐비넷) 제어, 외부 플랫폼(네이버 예약) 데이터 우회 동기화를 통합하는 것을 목표로 함
- **운행 환경:** 매장 내 윈도우 미니 PC (Windows 10/11 Pro 환경)를 키오스크 겸 제어 서버로 활용

---

## 2. 시스템 아키텍처 및 개발 환경 요구사항

### 권장 기술 스택

- **Front-end (Kiosk UI):** C# .NET (WPF) 또는 Electron + React/Vue (윈도우 Kiosk 모드 최적화 구동 가능해야 함)
- **Back-end & 제어 로직:** Node.js 또는 Python, C# (로컬 백그라운드 서비스 형태로 상시 구동)
- **Database:** 로컬 및 클라우드 동기화가 가능한 구조 (예: SQLite + 외부 메인 DB 동기화 구조)

### 연동 하드웨어 타겟

- **출입문 전자락:** 산업용 이엠락(EM Lock) 또는 데드볼트 (상태 감지 센서 포함 모델)
- **릴레이 컨트롤러:** USB 시리얼 4채널 릴레이 보드 (가상 COM 포트 통신)
- **결제 단말기:** 포스 연동용 IC 카드 리더기 및 영수증 프린터 (제조사제공 DLL/SDK 연동)

---

## 3. 핵심 기능별 요구사항 상세

### ① 네이버 예약 우회 연동 모듈 (공식 API 미사용)

네이버 공식 제휴 API를 사용하지 않고 예약을 동기화하는 모듈을 구축해야 합니다. 다음 두 가지 방식을 혼합하여 제안해 주십시오.

- **이메일 파싱 (메인):** 네이버 예약 확정 시 관리자 메일로 발송되는 알림 이메일을 IMAP/POP3 프로토콜로 실시간 파싱. 예약자명, 전화번호, 이용 상품, 기간 데이터를 추출하여 시스템 DB에 자동 등록
- **웹 크롤링/매크로 (서브):** Selenium 또는 Puppeteer를 활용하여 네이버 파트너센터에 주기적(또는 수동 동기화 버튼 클릭 시)으로 로그인하여 예약 리스트를 긁어와 검증하는 로직 구현 (세션 및 쿠키 관리 필수)

### ② 키오스크 UI 및 계약/결제 관리

- **UI/UX:** 현장 이용자를 위한 직관적인 대화면 UI. 창고 내부 도면(S/M/L 크기별)을 시각적으로 보여주고 비어 있는 캐비넷을 터치로 선택할 수 있는 배치도 뷰어 포함
- **현장 결제 연동:** 지정된 IC 카드 리더기 SDK를 연동하여 카드 결제/취소 프로세스 구현. 결제 성공 시 계약 데이터 생성 및 영수증(비밀번호 포함) 출력
- **계약 상태 관리:** 이용중, 만료예정, 정비필요, 공석 등 캐비넷 상태를 타임스탬프 기반으로 관리. 만료 전 자동 알림톡(카카오 비즈알림/SMS API) 발송 로직 포함

### ③ 출입문 및 하드웨어(릴레이) 제어

- **기본 제어:** 가상 COM 포트 시리얼 통신을 통해 USB 릴레이 보드로 On/Off Hex 명령어를 송신. 인증 성공 시 릴레이를 통제하여 전자락 전원을 3~5초간 차단 후 자동 잠금
- **인증 수단:** 알림톡으로 발송된 QR 코드 스캔(산업용 바코드 리더기 연동) 또는 고객 고유 PIN 번호(전화번호 기반) 입력
- **⭐ 필수 예외 처리 (오프라인 모드):** 매장 내 인터넷이 일시적으로 단절되어도 기존 계약자가 출입할 수 있도록 **Time-based OTP 알고리즘** 또는 로컬 DB 기반의 자체 인증 메커니즘을 반드시 구현할 것
- **안전 및 모니터링:**
  - 화재 수신기 접점 신호 인입 시 출입문 강제 개방(하드웨어 및 소프트웨어 연동)
  - 개폐 센서를 모니터링하여 '문 열림 지연(1분 이상)' 발생 시 관리자 알림 트리거
  - 관리자 페이지 내 '원격 문열기' 강제 제어 기능

### ④ 윈도우 무인 환경 최적화 (Kiosk 전용 설정)

- OS 부팅 시 키오스크 프로그램 자동 로그인 및 풀스크린 강제 구동
- Alt+F4, Ctrl+Alt+Del, 윈도우 키 등 사용자 임의 조작 방지를 위한 키 후킹 또는 Windows Assigned Access(할당된 액세스) 환경 가이드 제공
- **Watchdog 프로그램 구축:** 메인 키오스크 프로그램의 프로세스를 상시 감시하여, 프로그램이 다운되거나 강제 종료되었을 때 5초 이내에 자동 재구동시키는 별도의 경량 프로세스 동시 납품

## 4. 데이터베이스(DB) 스키마 요구사항 (필수 포함 테이블)

| 테이블 | 설명 |
|--------|------|
| `Users` | 회원 정보 (이름, 전화번호, 고유 PIN 등) |
| `Cabinets` | 창고 하드웨어 정보 (ID, 규격, 릴레이 채널 번호, 현재 상태) |
| `Contracts` | 계약 정보 (회원 ID, 캐비넷 ID, 시작일, 종료일, 상태) |
| `Payments` | 결제 이력 (계약 ID, 금액, PG 승인번호, 결제시각, 상태) |
| `AccessLogs` | 출입 기록 (일시, 회원 ID, 인증 방식, 성공/실패 여부) |
